# Website Builder System - Complete Flow Explanation

## 🏗️ OVERVIEW
This is a dynamic website builder that allows users to edit sections of a webpage in real-time. The system uses React, Redux Toolkit, and Next.js to provide an intuitive editing experience.

---

## 📋 CORE COMPONENTS

### 1. **Redux Store (builderSlice.ts)**
- **Purpose**: Central state management for all builder functionality
- **Key State Properties**:
  - `isBuilderMode`: Toggle builder mode on/off
  - `sections`: Array of all website sections (hero, banner, features, etc.)
  - `activeSection`: Currently selected section for editing
  - `isPreviewMode`: Preview mode toggle

### 2. **HomePage (index.tsx)**
- **Purpose**: Main page that renders all sections
- **Key Change**: Now renders ALL sections from Redux (including hero) in correct order
- **Logic**: Sorts sections by `order` property and renders them sequentially
- **Flow**: No more separate static Hero - everything comes from Redux state

### 3. **Hero.tsx (Modified)**
- **Purpose**: The hero section component that users see
- **Hybrid Behavior**: Merges static content from homeSlice with dynamic content from builderSlice
- **Content Logic**: 
  ```typescript
  const heroContent = {
    ...staticHeroContent,           // Base content from homeSlice
    ...(dynamicHeroSections[0]?.content || {})  // Override with edited content
  };
  ```
- **Responsive Design**: Added width constraints and proper text wrapping
- **Key Changes**: 
  - Container: `max-w-xl lg:max-w-2xl xl:max-w-3xl`
  - Title: Smaller responsive sizes with `break-words hyphens-auto`
  - Subtitle: `max-w-full` with proper wrapping
  - Spacing: Reduced from `pt-[80px]` to `pt-[50px]`

### 4. **WebsiteBuilder.tsx**
- **Purpose**: Builder UI with section list and editor panels
- **Auto-opens**: Automatically opens hero editor when builder mode activates
- **Mobile Support**: Different layouts for mobile vs desktop
- **Key Actions**: Add, edit, delete, reorder sections

### 5. **HeroEditor.tsx**
- **Purpose**: Editor panel for hero section content
- **Image Upload**: Uses FileReader to convert to base64 (immediate display)
- **Field Updates**: Real-time updates to Redux store
- **Content Management**: Only updates section-specific content, not global state
- **Key Fix**: Removed dependency on homeSlice, uses only builder content

## 🔄 COMPLETE DATA FLOW

### **Initial Page Load**
1. Redux store initializes with default sections from `builderSlice.ts`
2. HomePage renders all sections in order (including default hero)
3. Hero.tsx receives static content (no dynamic content yet)
4. WebsiteBuilder is hidden (builderMode = false)

### **Activating Builder Mode**
1. User clicks builder button → `toggleBuilderMode()` action dispatched
2. Redux state: `isBuilderMode = true`
3. WebsiteBuilder appears with section list
4. `useEffect` auto-opens first hero section editor
5. Section list and editor panel appear simultaneously

### **Adding New Hero Section**
1. User clicks "Hero" button in SectionList
2. `addSectionAndSetActive` action creates new section:
   ```typescript
   {
     id: 'hero-1712345678900',  // Unique ID
     type: 'hero',
     name: 'Hero Section',
     visible: true,
     order: 7,  // Next available order
     content: getDefaultContent('hero')  // Default hero content
   }
   ```
3. New section is set as active section
4. HeroEditor opens with empty/default content
5. User edits content → updates builderSlice
6. HomePage re-renders with new section in correct position

### **Editing Existing Hero Section**
1. User clicks edit button on existing hero section
2. `setActiveSection('hero-1')` dispatched
3. WebsiteBuilder determines which hero section is being edited
4. HeroEditor receives current section content and renders input fields
5. User makes changes → `updateField()` called
6. Creates new content object with updated field
7. Calls `updateSectionContent()` action
8. Redux store updated → HomePage re-renders with merged content

### **Content Merging in Hero.tsx**
```typescript
// Always shows Hero component, but content is dynamic
const staticHeroContent = useAppSelector(state => state.home.heroContent);
const dynamicHeroSections = useAppSelector(state => state.builder.sections.filter(s => s.type === 'hero'));

const heroContent = {
    ...staticHeroContent,           // Base content from homeSlice
    ...(dynamicHeroSections[0]?.content || {})  // Override with edited content
};

// Result: Only edited fields change, unedited fields keep original values
```

### **Image Upload System**
**Before**: Created fake URL paths like `/images/filename.jpg`
**After**: 
```javascript
const handleImageUpload = (sectionId, imageType) => {
  const input = document.createElement('input');
  input.type = 'file';
  input.accept = 'image/*';
  input.onchange = (e) => {
    const file = e.target.files[0];
    if (file) {
      const reader = new FileReader();
      reader.onload = (e) => {
        const result = e.target.result;  // base64 data URL
        updateField(imageType, result);  // Immediate update
      };
      reader.readAsDataURL(file);
    }
  };
  input.click();
};
```

### **Drag and Drop Reordering**
**Before**: Sections rendered separately, reordering didn't affect output
**After**:
1. User drags section in SectionList
2. `reorderSections({ fromIndex: 2, toIndex: 0 })` action dispatched
3. Redux store updates section `order` properties
4. HomePage re-sorts sections by new order
5. Visual output changes immediately

##  Banner System - Complete Architecture & Flow

###  Overview
The Banner system is a sophisticated component architecture that handles multiple section types (SecondSection, ThirdSection, and DynamicBanner) with dynamic icon rendering, image tilt effects, and real-time editing capabilities.

###  Core Components

#### 1. **DynamicBanner.tsx** (Universal Banner Component)
- **Purpose**: Universal banner component that can render any section type
- **Dual Store Support**: Reads from both `builderSlice` and `bannerSlice`
- **Smart Icon Rendering**: Uses SVG icons for second/third sections, Lucide icons for others
- **Image Tilt Effects**: Implements hover-based image tilting with animations

#### 2. **SecondSection.tsx** (Specialized Component)
- **Purpose**: Dedicated component for "second" type sections
- **Icon Mapping**: Uses `list`, `live`, `HeroImage` SVG icons
- **Left Image Layout**: Image always positioned on right side
- **Tilt Effect**: Always tilts left on hover (since image is on right)

#### 3. **ThirdSection.tsx** (Specialized Component)
- **Purpose**: Dedicated component for "third" type sections  
- **Icon Mapping**: Uses `HeroImage`, `Audience`, `Gift`, `Sword` SVG icons
- **Left Image Layout**: Image always positioned on right side
- **Tilt Effect**: Always tilts left on hover

#### 4. **BannerEditor.tsx** (Editor Component)
- **Purpose**: Universal editor for all banner sections
- **Icon Preview**: Shows correct SVG icon previews for second/third sections
- **Real-time Updates**: Immediate content updates via Redux
- **Feature Management**: Add/edit/delete features with icon support

###  Data Flow Architecture

#### **Section Routing Logic** (index.tsx)
```typescript
switch (section.type) {
  case 'banner':
  case 'live-streaming':

    return <SecondSection key={section.id} sectionId={section.id} />;
  case 'third':
```

###  Icon Rendering System

#### **DynamicBanner Icon Logic**
```typescript
const renderIcon = (iconName, className, sectionType, featureIndex) => {
  const isSecondSection = sectionType === 'second' || sectionId.includes('second');
  const isThirdSection = sectionType === 'third' || sectionId.includes('third');
  
  if (isSecondSection) {
    if (featureIndex === 0) return <Image src={list} alt="List icon" />;
    if (featureIndex === 1) return <Image src={live} alt="Live icon" />;
    if (featureIndex === 2) return <Image src={HeroImage} alt="Hero icon" />;
    return <Image src={list} alt="List icon" />; // Fallback
  }
  
  if (isThirdSection) {
    if (featureIndex === 0) return <Image src={HeroImage} alt="Hero icon" />;
    if (featureIndex === 1) return <Image src={Gift} alt="Audience icon" />;
    if (featureIndex === 2) return <Image src={Audience} alt="Gift icon" />;
    if (featureIndex === 3) return <Image src={Sword} alt="Sword icon" />;
    return <Image src={HeroImage} alt="Hero icon" />; // Fallback
  }
  
  // Fallback to Lucide icons for other sections
  switch (iconName) {
    case 'Video': return <Video {...iconProps} />;
    case 'MessageCircle': return <MessageCircle {...iconProps} />;
    // ... other Lucide icons
  }
};
```

## Admin Panel System - Complete Architecture & Flow

### Overview
The Admin Panel system is a sophisticated tabbed interface system that allows users to create, edit, and manage admin panel sections with multiple tabs. It supports drag-and-drop reordering, layout customization, and real-time content updates.

### Core Components

#### 1. **AdminEditor.tsx** (Main Editor Component)
- **Purpose**: Universal editor for admin panel sections
- **Tab Management**: Create, edit, delete, and reorder tabs (max 6 tabs)
- **Layout Control**: Toggle between "Image Left" and "Image Right" layouts
- **Color Styling**: Section-level color customization for all text elements
- **Drag & Drop**: Visual tab reordering with feedback

#### 2. **FifthSection.tsx** (Display Component)
- **Purpose**: Renders individual admin panel sections with tabs
- **Dynamic Layout**: Supports left/right image positioning
- **Color Support**: Applies section-level colors to all elements
- **Tab Sorting**: Displays tabs in correct order based on `order` property

#### 3. **DynamicAdmin.tsx** (Aggregator Component)
- **Purpose**: Aggregates tabs from all fifth sections
- **Unified Display**: Shows all admin tabs in a single interface
- **Layout Inheritance**: Uses layout from first fifth section
- **Tab Filtering**: Only shows visible tabs from all sections

### Data Flow Architecture

#### **Section Creation Flow**
1. **User clicks "Admin Panel" button** in SectionList
2. **`addSectionAndSetActive` action dispatched** with type `'fifth'`
3. **New section created** with unique ID: `fifth-${Date.now()}`
4. **Default content applied** with 2 demo tabs and section colors
5. **AdminEditor opens** automatically for the new section
6. **User edits tabs/content** via AdminEditor
7. **Changes saved to builderSlice** and reflected immediately

```typescript
const getAllTabs = (): AdminTabContent[] => {
  const tabs: AdminTabContent[] = [];
  
  const fifthSections = builderSections.filter(s => s.type === 'fifth');
  fifthSections.forEach(fifthSection => {
    if (fifthSection?.content?.tabs && Array.isArray(fifthSection.content.tabs)) {
      const visibleTabs = fifthSection.content.tabs.filter(t => t.visible !== false);
      tabs.push(...visibleTabs);
    }
  });
  
  return tabs;
};
```
